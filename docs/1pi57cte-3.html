『修复』做了个GFWList PAC最新下载地址，可用于更新 SSR PC客户端的PAC

Shadowsocks Toyo 1年前 (2017-07-30) 225评论
广告
本文最后更新于 2018年10月16日 13:07 可能会因为没有更新而失效。如已失效或需要修正，请留言！

考虑到ShadowsocksR因为暂停更新项目删除，导致客户端无法直接更新GFWList PAC文件，所以我写了个脚本自动把 gfwlist 转换为ShadowsocksR客户端可用的 pac.txt 文件。

pac.txt 文件会每天定时生成一次，在 pac.txt 文件的顶部写的有最后更新时间。

大家只需要每隔一段时间下载这个文件并覆盖 ShadowsocksR.exe 同目录下的 pac.txt 文件即可。

（如果找不到，那么点击 右键SSR客户端托盘图标 - PAC - 编辑本地PAC文件...）。

GFWList项目：https://github.com/gfwlist/gfwlist

GFWList地址：https://raw.githubusercontent.com/gfwlist/gfwlist/master/gfwlist.txt

GFWList PAC地址：https://softsmirror.cf/Other/pac.txt（右键 链接另存为... 每天更新但需要挂代理下载）

GFWList PAC地址：https://raw.githubusercontent.com/ToyoDAdoubiBackup/doubi/master/other/pac.txt（右键 链接另存为...）

脚本开源地址：Github

注意：此 GFWList pac.txt 生成脚本只是提供给需要的人学习研究用的，普通用户并不需要用这个脚本，只需要下载 pac.txt 覆盖即可。

下载覆盖 pac.txt 前，请先关闭ShadowsocksR客户端。

注意事项

  1. 因为 gfwlist 更新频率不高，往往一个月也没更新多少条内容，所以不需要频繁更新 pac.txt 文件，偶尔更新一次就行了。
  2. 如果使用 Proxy SwitchyOmega 类代理管理扩展的情景模式时，只需要使用 GFWList 的地址即可，不需要用我的PAC文件。
  3. 本PAC并不接受提交被墙域名，本PAC只是转换GFWList的网站规则，所以要提交请前往 GFWList Github Issues 提交。

注意：ShadowsocksR PC客户端最后版本为 4.7.0，而其他的版本比如 4.8.0 皆为第三方修改版，只是拿着开源的披着SSR的外衣罢了。

      石板2017-07-31 17:54 回复
      • PAC（或者说所有系统代理模式）是浏览器进行判断访问网站数据是否走代理（进入SSR客户端），而代理规则则是判断 进入SSR客户端的 数据 是直连还是继续走代理（发往SSR服务端）。
        代理规则 - 绕过局域网和大陆：分析进入SSR的流量，判断目标IP地址，IP是在国外，那么继续走代理。
        代理规则 - 全局：不分析进入SSR的流量，直接继续走代理。
        在系统代理模式：PAC+代理规则：绕过局域网和大陆 下，访问XXX网站，会判断XXX网站在PAC规则中走代理，如果走代理，那么在判断XXX网站IP是否是国外IP，如果是就继续走代理。
        你说G+只能打开一半，你可以在打开G+前，按F12打开浏览器的开发者工具，查看网站链接加载情况，看哪些域名元素没有正常加载。
        你说的情况，可能是G+里部分链接在PAC或者代理规则初，被过滤下来了，可能是因为 PAC或代理规则不够完整（这是不可避免的），导致加载失败的网络链接没有走代理，而是直连，而这些域名直连无法访问，于是可能出现你说的一半加载，一般加载失败的情况。
        一般来说，PAC+全局，全局+绕过局域网和大陆，这两种配合最好。
